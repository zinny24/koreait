1. 응집도
④ 우연적 응집도 - ㉣모듈 내부의 각 구성 요소들이 연관이 없을 경우이다.
② 순차적 응집도 - ㉠ 모듈 내에서 한 활동으로부터 나온 출력값을 다른 활동의 입력값으로 사용하는 경우이다.
③ 절차적 응집도 - ㉢ 모듈이 다수의 관련 기능을 가질 때 모듈 안의 구성 요소들이 그 기능을 순차적으로 수행할 경우이다.
④ 기능적 응집도 - ㉡ 모듈 내부의 모든 기능이 단일한 목적을 위해 수행되는 경우이다.


2. 배치 프로그램이란 사용자와의 상호 작용 없이 일련의 작업들을 작업 단위로 묶어 정기적으로 반복 수행하거나 정해진 규칙에 따라 일괄 처리하는 것이다.

3. 테스트 케이스의 구성 요소에는 식별자, 테스트 항목, 입력 명세, 출력 명세, 환경 설정 등이 있다.


4. 개발을 위해 사용되는 소프트웨어의 종류 중 구현 도구 Eclipse, visual Studio Code, IntelliJ가 있다

5. 개발 소프트웨어 환경도 개발 하드웨어 환경과 마찬가지로 운영환경과 동일한 구조로 구성하는 것이 원칙이며, 개발용 소프트웨어 환경을 구축하기 위해서는 다음과 같은 유형의 소프트웨어 구성을 고려하여야 한다


6. 테스트 케이스란 요구 사항을 준수하는지 검증하기 위하여 테스트 조건, 입력값, 예상출력값 및 수행한 결과 등 테스트 조건을 명세한 것이다

7. 형상관리 도구에는 SVN, GIT이 있다.

1. 통합구현 필요한 요구 사항 분석 시 입력물 중 시스템 구성도에는 시스템 소프트웨어 구성, 하드웨어, 송수신 시스템의 네트워크이 있다.

2. 통합구현 개념
① 송신 시스템과 모듈 : 전송하고자 하는 데이터를 생성하여 필요에 따라 변환 후 송신하는 송신 모듈과 데이터 생성 및 송신 상태를 모니터링하는 기능으로 구성된다
② 연계 데이터 : 송수신되는 데이터로 의미를 갖는 속성, 길이, 타입 등이 포함된다. 
③ 중계 시스템 : 주로 외부 시스템 간의 연계 시에 적용되는 아키텍처로, 내외부 구간의 분리로 보안성이 강화되고, 인터넷 망(Internet)과 인트라넷 망(Intranet)을 연결할 수도 있다.
④ 네트워크 : 송신 시스템과 수신 시스템, 송신 시스템과 중계 시스템, 중계 시스템과 수신 시스템을 연결해주는 통신망을 말한다.


3. 델파이 기법: 통합 구현 및 연계 전문가, 시스템 아키텍처, 업무 전문가 등 각 분야 전문가로 부터 연계 데이터 및 사용자 요구 사항 식별


4. 연계 메커니즘 구성 요소별로 송신 시스템 운영시 사용하는 파일 형식에는 XML, CSV, TEXT이 있다.


5. 선형 검색은 처음부터 끝까지 하나씩 순서대로 비교하며 원하는 값을 찾아내는 검색을 말한다.


6. 연계 데이터 보안 중 데이터 보안에 대한 구현 방법에는 연계 메커니즘에서 데이터 암·복호화 처리 절차, 암·복호화 적용 대상 선정, 암호화 알고리즘 적용이 있다.
