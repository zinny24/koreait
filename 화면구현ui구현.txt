1. 화면설계
  (1) 피드백 : 화면 구현을 할때 진행 상황을 표시하고 표시된 내용을
                 해석할 수 있도록 도와주는 행위
  (2) 프로토타입(샘플) : 고객이 요청한 사항을 정확하게 반영해서 작업을 했나?
  (3) 웹의 3요소 : 웹 표준성, 웹 접근성, 웹 호환성
  (4) 화면 설계의 종류 : 스토리보드, 와이어 프레임, Mock up
  (5) W3C : 웹 표준을 개발하고 장려하는 국제 표준 기구
  (6) 시멘틱 태그 : div -> header, main, footer, section, article, aside...
       (html)
  (7) css 선택자 : id(#), class(.), 태그선택자(h1, p, nav...)
                       공용선택자(유니버셜 태그) : *

  (8) 유효성 검사 : 웹 표준에 맞게 태그를 작성했는지 확인...
       ex) w3c 유효성 검색

2 .UI구현
  (1) 휴리스틱 평가 : UI 구현 시 규칙을 만들어 놓고 그 규칙을 얼마나 잘 지키는 지 평가하는 것
  (2) 웹 사이트 개발에 필요한 기술
      - JSP, ASP, .NET, PHP, html, css, javascript
      - Active X

   (3) UI 구현 시 선호도 평가의 내용 3가지 : 점수, 순위, 태도

   (4) UI/UX 구현 : 화면 디자인 & 사용자 경험

   (5) html 구조
      
   (6) 프로토타입 체크리스트 구성 요소 : 유저시나리오, 사이트맵, flow chart
